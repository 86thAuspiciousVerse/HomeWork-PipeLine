# 课程设计要求：城市空气质量预测

## 题目

基于中国环境监测总站的全国空气质量监测数据，对指定城市的空气质量进行趋势分析和预测建模。

## 数据来源

- 中国环境监测总站发布的全国空气质量小时级监测数据
- 数据已提供在 `data/raw/` 目录下（每日一个 CSV 文件）
- 站点元数据见 `data/站点列表.csv`

## 数据格式说明

原始数据为**宽表格式**：每行 = 日期 + 小时 + 污染物类型，列为各监测站点编号。包含 AQI、PM2.5、PM10、SO2、NO2、O3、CO 等指标。**使用时需要将宽表转置为长表**。

## 要求

1. **数据清洗**：宽表转长表、缺失值处理（前向填充 + 连续缺失 >6h 标记）、分块读取避免内存溢出
2. **特征工程**：构建时间特征（年/月/日/星期/季节）、滞后特征、滚动统计特征
3. **趋势分析**：多时间尺度趋势分解（年际、季节、周）
4. **预测建模**：时间序列预测模型，预测未来 24-72 小时的 PM2.5/AQI
5. **模型服务化**：FastAPI 接口（含 Swagger 文档）
6. **可视化仪表盘**：交互式 HTML Dashboard（Plotly 自包含模式）

## 技术栈要求

- Python 3.10+
- pandas、numpy、scikit-learn、Plotly、FastAPI
- 时间序列模型可选：Prophet、XGBoost、Linear Regression

## 关键约束

- **严禁 `train_test_split` 随机切分**，必须按时间顺序切片（70/15/15）
- 不需要处理全部站点，选 1-3 个数据完整度 >90% 的代表城市
- PEP8 规范 + `requirements.txt` 版本锁定

## 交付物

- 交互式 HTML Dashboard（自包含，双击打开）
- FastAPI 服务（`app.py`，含 `/docs` Swagger 文档）
- 完整可复现代码（含 `requirements.txt`）
